# Contributing

Thanks for your interest in `spatial-cluster-compare`!

This project is maintained solo, as part of a personal bioinformatics
toolkit ([MACSima_Spatial-Omics-Pipeline](https://github.com/mathisbouvet/MACSima_Spatial-Omics-Pipeline)).
To keep the codebase consistent and maintainable, contributions are handled
as follows:

## Reporting a bug

Please [open an issue](../../issues/new) with:

- A short description of the problem
- A minimal reproducible example (a small CSV sample or code snippet)
- The package version (`pip show spatial-cluster-compare`) and Python version

## Suggesting a feature or improvement

Please [open an issue](../../issues/new) describing:

- The use case / problem you're trying to solve
- Why the existing API (`compare_clusters`, step-by-step functions) doesn't
  already cover it

This is the best way to start — it lets us discuss the approach before any
code is written.

## Pull requests

Pull requests are currently accepted **by invitation only**. If you've
opened an issue and we agree it's worth implementing, I may ask you to open
a PR, or pick it up myself. Unsolicited PRs may be closed without review,
even if the contribution is valuable — this isn't a reflection of its
quality, just a way to keep scope and review effort manageable for a
single-maintainer project.

## Questions

For anything else (usage questions, clarifications on the methodology),
feel free to open a [discussion](../../discussions) or an issue.
